微博超话自动签到

主要由DeepSeek完成

写在最前

如有志同道合者，对细节流程不清楚的，欢迎留言交流。

代码内容请自行研究，笔者纯DeepSeek，只是做了创建仓库、添加环境变量、复制代码、执行测试流程、判断是否达到预期效果等工作

一、项目目标

在 GitHub 上部署一个定时任务，每天北京时间 8:00 自动登录微博并完成超话签到，签到失败时发送邮件通知。

二、技术方案演进

1. 初始方案：模拟浏览器点击

思路：使用 Puppeteer (Node.js) 启动无头浏览器，自动访问超话页面并点击签到按钮。

问题：Cookie 获取复杂、依赖安装较多，用户选择了 Python 方案。

2. Python 直调 API 尝试

思路：直接调用微博签到接口 https://i.huati.weibo.com/aj/super/checkin。

遇到问题：

  • ❌ i.huati.weibo.com 域名在 GitHub Actions 环境连接超时
  • ❌ 本地测试时也频繁 SSL 错误和连接拒绝
  • ❌ 换用 https://weibo.com/aj/immobile/super/checkin 返回 系统繁忙，但实际未签到（假成功）

结论：API 直调在 GitHub Actions 网络环境下不可靠，且无法准确判断签到结果。

3. 最终方案：Selenium 浏览器自动化 ✅

思路：使用 Selenium + Chrome Headless 在 GitHub Actions 中模拟真实浏览器操作。

关键实现：

  • 通过 Cookie 保持登录状态
  • 精确选择签到按钮：.btn_bed .W_btn_b[action-type="widget_take"]
  • 点击前判断按钮文字，点击后验证是否变为“已签到”
  • 签到结果通过邮件通知

优点：

  • ✅ 稳定可靠，不受微博接口变化影响
  • ✅ 能准确判断签到成功与否
  • ✅ 维护成本低

三、最终文件结构

核心文件内容

sign.yml 核心逻辑

sign.py 核心逻辑

  1. 解析环境变量中的 Cookie
  2. 启动 Chrome 无头浏览器
  3. 注入 Cookie 实现登录
  4. 访问超话页面，精确定位签到按钮
  5. 点击签到，验证结果
  6. 发送邮件通知

四、配置要点

GitHub Secrets 配置

Secret 说明 示例
WEIBO_COOKIE 微博登录 Cookie（含 SUB 字段） SUB=_2A25Eu...; SUBP=...
EMAIL_USER 发件邮箱 123456@qq.com
EMAIL_PASS 邮箱授权码（非登录密码） abcdefghijklmn
NOTIFY_EMAIL 收件邮箱 可留空，默认用发件邮箱

获取微博 Cookie

  1. 浏览器登录 https://weibo.com
  2. F12 → Network → 刷新页面
  3. 找到任意请求，复制 Request Headers 中的 Cookie 值
  4. 特别确保包含 SUB 字段

获取邮箱授权码

  • QQ邮箱：设置 → 账户 → POP3/IMAP服务 → 生成授权码
  • 163邮箱：设置 → POP3/SMTP/IMAP → 开启服务并获取授权码

五、最终验证结果

脚本能够准确判断未签到状态，完成点击，并确认签到成功。

六、资源消耗评估

项目 免费额度 实际使用 占比
GitHub Actions 时长 2000分钟/月 约 90分钟/月 4.5%
存储空间 500MB < 1MB 忽略

完全在免费额度内，无需担心。

七、维护建议

  1. Cookie 有效期约 1 个月，失效后重新获取并更新 Secret
  2. 若微博页面改版导致选择器失效，调整 sign.py 中的 CSS 选择器即可
  3. 可登录 GitHub Actions 查看历史运行日志，确认每日签到状态

八、总结

项目最终采用 Selenium 浏览器自动化方案，成功实现了：

  • ✅ 每天北京时间 8:00 自动签到
  • ✅ 准确判断签到状态
  • ✅ 失败时邮件通知
  • ✅ 完全在 GitHub 免费额度内运行

整个过程中遇到的 API 不通、假成功、按钮定位错误等问题均已解决，当前方案稳定可靠，可长期使用。
