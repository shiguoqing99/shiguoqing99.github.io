基于ipv6和dynv6的网站部署

写在最前

时效性

本篇实际部署实际是在4月30日完成，中间鸽了好久到今天（5月7日）才写完，一些具体的细节可能记不太清了或者可能有些错误请批评指正、互相学习

优缺点

使用公网ipv6进行访问，不需要申请ipv4备案等等，一般家用宽带都会有

但是电脑上必须开启TCP/IPv6才能访问，不开启完全无法访问，一般电脑都支持，但是否开启不一定，手机访问通常不受影响

以下正文

前言

前几天把NAS搞好了，这几天看了看外网访问。暂时常规的一级域名+ipv4+ICP备案的路子暂时走不通，于是打算换个路子，尝试一下ipv6+dynv6的方案。

弯路

简单概述一下走过的弯路

  1. 使用cloudns+cloudfare，可以申请域名，但是只能是二级域名，使用cloudns免费用户不允许使用二级域名
  2. 使用路由器端口转发的方案，找了半天也没找到ipv6的防火墙
  3. …

部署过程

1. 获取IPv6地址

在Ubuntu服务器上得到IPv6地址

得到

2. 获取域名

注册登录

登录dynv6官网https://dynv6.com/，点击右上角sign up注册

输入注册邮箱、密码和重复密码，点击下方sign up

此时注册邮箱会收到一封确认邮件Confirmation instructions，点击邮件中的Confirm my account按钮，即可完成注册。这一步骤需要启用代（科）理（学）上网，否则无法弹出确认的验证框

使用注册邮箱登录。

创建新域名

进入首页，点击Create new Zone自己起一个名字

Only characters a-z 0-9 - are allowed, 5-20 characters. 5-20个字符，只允许字母、数字和-

选择二级域名，根据自己喜好选择即可

如果有公网ipv4地址，可以填写，没有的话不填，填写ipv6地址，填写上述获得的ipv6地址，点击Create Zone

点击自己创建的域名，点击Record，Type选择AAAA，Name是自己填的，Data是ipv6地址

域名获取完成，域名与ipv6地址绑定完成

3. 配置Web服务器

一开始我用的Nginx，但是没有SSL证书，所以就换成了Caddy（没有SSL是http连接，有SSL是https连接，http会提示不安全）

以下源于deepseek

因为 80 端口（HTTP）和 443 端口（HTTPS）同一时间只能被一个程序占用。

Nginx 和 Caddy 都是 Web 服务器，都要抢这两个端口。

解决方案：

方案 做法 适用场景
二选一 停掉 Nginx，只用 Caddy 需要自动 HTTPS
Caddy 做“门卫” Nginx 改成其他端口（如 8080），Caddy 反向代理到 Nginx 想保留 Nginx 功能
Nginx 做“门卫” Caddy 改成其他端口，Nginx 反向代理到 Caddy 熟悉 Nginx 配置

最简单的：不需要 Nginx 的话就直接停掉它。Caddy 自带了反向代理功能，Nginx 能做的事 Caddy 基本都能做。
