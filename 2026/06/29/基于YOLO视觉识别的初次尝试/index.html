基于YOLO视觉识别的初次尝试

发表于2026-06-29|发生于2026-06-30|指间世界
|浏览量:

前言

最近开始做一个视觉识别、视觉追踪相关的项目，开始学习测试相关内容

工具

硬件

  1. 计算机（任意系统均可，这里使用 Windows）
  2. USB接口摄像头（因本设备没有摄像头，使用手机摄像头+ivcam平替）
  3. 识别目标，这里在纸上画了个五角星

软件

  1. VSCode
  2. Miniconda3-py39
  3. ultralytics
  4. 代理工具（需要大量科学上网，否则会很慢甚至无法进行）

配置环境

下载安装Miniconda3-py39

下载网站（清华源）
https://mirrors.tuna.tsinghua.edu.cn/anaconda/miniconda/

找到Miniconda3-py39_22.11.1-1-Windows-x86_64.exe下载（根据操作系统选择版本，软件版本和Python版本最好不要太新）

安装位置自定义，笔者装在D盘，推荐勾选Add Miniconda3 to my PATH environment variable，添加到系统环境，后面省事

创建虚拟环境及配置

打开cmd或者Anaconda Prompt（不要使用Powershell），用 Conda 创建一个名为yolov8的新虚拟环境，并安装 Python 3.9 版本（推荐指定版本）

激活环境

配置国内下载源

安装PyTorch

官网https://pytorch.org/get-started/previous-versions/

笔者电脑为Intel Arc显卡，安装xpu版本

英伟达（nvida）显卡版本在官网选择对应的PyTorch版本

进入Python环境进行测试

环境配置完成

下载安装ultralytics

ultralytics 这个 Python 库，不仅支持 YOLOv8，还支持 YOLOv5、YOLOv9、YOLOv10、YOLO11 等模型的训练、推理和部署。

地址https://github.com/ultralytics/ultralytics，code -> Download zip

解压，在vscode内进入yolov8环境，安装yolov8

使用自带图片进行测试

使用yolov8n模型对ultralytics/assets/bus.jpg进行识别检测

结果

在runs\detect\predict查看可视化检测结果

训练自定义模型

下载labelimg

labelimg是用来图标标签标注的工具

安装完输入labelimg打开软件

整理素材

对检测目标进行多方位、多角度、各种光线、状态拍照取景，也可以拍视频后抽帧，图片数量越多越好，不同的状态越多训练效果越好（以数百张为单位）

图片标注

基础配置

勾选View -> Auto Save Mode，标注完自动保存提高速度

更换模型

默认打开可能是PascalVOC，更换为YOLO

导入导出路径

Open Dir打开图片素材所在文件夹，第二个弹出框Save annotations to the directory选择标签保存路径

标注图片

打开图片后快捷键W框选目标，填写标签信息，可以选择多个目标并设置不同标签，快捷键D下一张图片，快捷键A上一张图片，如此完成所有图片的标注。（最耗费时间的一步）

整理图片和标签

  1. 在ultralytics-main根目录创建datasets文件夹
  2. datasets文件夹下创建images和labels文件夹
  3. images和labels文件夹下分别创建train和val文件夹
  4. 将素材图片放到images文件夹，train的占80%左右，val占20%左右
  5. 将txt标签文件放到labels文件夹，train的占80%左右，val占20%左右
  6. classes.txt放在labels目录下
  7. 图片名和标签名对应 说明
  8. bvn训练库文件夹，可以不设置
  9. images存放所有图片
  10. labels存放所有标签
  11. train训练集，训练模型用，占比80%左右
  12. val验证集，验证模型用，占比20%左右
  13. 图片名和标签名对应，

配置描述文件

从ultralytics\cfg\datasets中任意复制一份yaml文件到根目录，重命名为yolo-bvn（任意，可选）

  1. pathdataset根目录路径，这里使用bvn
  2. train训练集目录路径，这里使用images/train
  3. val验证集目录路径，这里使用images/val
  4. test 测试集，暂时不写
  5. names标签类名，根据classes.txt内容填写

资源文件、标注文件、配置文件搞定后，开始训练

训练结果

训练完成

30轮训练已完成，耗时0.383小时（约23分钟）。

模型权重已保存

  • 优化器已从以下文件中剥离（精简模型，方便部署）：
    • D:\AIot\ultralytics-main\runs\detect\train\weights\last.pt（最后一轮模型），大小 6.5MB
    • D:\AIot\ultralytics-main\runs\detect\train\weights\best.pt（最佳模型），大小 6.5MB

开始验证（评估最佳模型性能）

  • 当前环境：Ultralytics 8.4.83，Python 3.9.25，PyTorch 2.7.1+xpu，实际使用CPU（Intel Core Ultra 5 125H）
  • 模型概况（已融合）：共73层，315.19万参数，0梯度（冻结），计算量8.7 GFLOPs

验证集评估结果

  • 评估数据：27张图片，25个标注目标（均为“X”类别）
  • 性能指标：
    • Box(P)（边界框精确率）：1 → 100%（预测为正例的样本中，真实为正例的比例）
    • R（召回率）：0.997 → 99.7%（真实正例中被正确检测出来的比例）
    • mAP50（IoU=0.5时的平均精度）：0.995 → 99.5%
    • mAP50-95（从IoU=0.5到0.95的平均精度）：0.726 → 72.6%（更严格的指标，反映模型定位精度）

处理速度

  • 预处理：0.4毫秒/图
  • 推理：30.3毫秒/图（约33 FPS，帧率）
  • 后处理：0.3毫秒/图

结果保存位置

所有结果已保存到：D:\AIot\ultralytics-main\runs\detect\train

默认配置覆盖（可选）

如果配置内容一致，可以加训练参数的配置文件复制出来，在配置文件中固定训练参数，即可避免每次训练都写很长的参数

项目根目录下，会生成一个default_copy.yaml文件，修改里面的参数即默认训练参数

开始训练yolo cfg-default_copy.yaml
