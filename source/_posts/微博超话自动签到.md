## 一、项目目标

在 GitHub 上部署一个定时任务，每天北京时间 8:00 自动登录微博并完成超话签到，签到失败时发送邮件通知。

---

## 二、技术方案演进

### 1. 初始方案：模拟浏览器点击

**思路**：使用 Puppeteer (Node.js) 启动无头浏览器，自动访问超话页面并点击签到按钮。

**问题**：Cookie 获取复杂、依赖安装较多，用户选择了 Python 方案。

---

### 2. Python 直调 API 尝试

**思路**：直接调用微博签到接口 `https://i.huati.weibo.com/aj/super/checkin`。

**遇到问题**：
- ❌ `i.huati.weibo.com` 域名在 GitHub Actions 环境连接超时
- ❌ 本地测试时也频繁 SSL 错误和连接拒绝
- ❌ 换用 `https://weibo.com/aj/immobile/super/checkin` 返回 `系统繁忙`，但实际未签到（假成功）

**结论**：API 直调在 GitHub Actions 网络环境下不可靠，且无法准确判断签到结果。

---

### 3. 最终方案：Selenium 浏览器自动化 ✅

**思路**：使用 Selenium + Chrome Headless 在 GitHub Actions 中模拟真实浏览器操作。

**关键实现**：
- 通过 Cookie 保持登录状态
- 精确选择签到按钮：`.btn_bed .W_btn_b[action-type="widget_take"]`
- 点击前判断按钮文字，点击后验证是否变为“已签到”
- 签到结果通过邮件通知

**优点**：
- ✅ 稳定可靠，不受微博接口变化影响
- ✅ 能准确判断签到成功与否
- ✅ 维护成本低

### 核心文件内容

#### `sign.yml` 核心逻辑

```yaml
name: 微博签到
on:
  schedule:
    - cron: '0 0 * * *'          # 每天北京时间 8:00
  workflow_dispatch:               # 支持手动触发
jobs:
  sign:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: '3.11' }
      - run: sudo apt-get install -y chromium-browser
      - run: pip install selenium
      - run: python sign.py
        env:
          WEIBO_COOKIE: ${{ secrets.WEIBO_COOKIE }}
          EMAIL_USER: ${{ secrets.EMAIL_USER }}
          EMAIL_PASS: ${{ secrets.EMAIL_PASS }}
          NOTIFY_EMAIL: ${{ secrets.NOTIFY_EMAIL }}
```

#### `sign.py` 核心逻辑
1. 解析环境变量中的 Cookie
2. 启动 Chrome 无头浏览器
3. 注入 Cookie 实现登录
4. 访问超话页面，精确定位签到按钮
5. 点击签到，验证结果
6. 发送邮件通知

```python
#!/usr/bin/env python3
import os
import time
import smtplib
from email.mime.text import MIMEText
from datetime import datetime
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.webdriver.chrome.options import Options

COOKIE_STR = os.environ.get('WEIBO_COOKIE', '')
EMAIL_USER = os.environ.get('EMAIL_USER', '')
EMAIL_PASS = os.environ.get('EMAIL_PASS', '')
NOTIFY_EMAIL = os.environ.get('NOTIFY_EMAIL', '')

TOPIC_URL = 'https://weibo.com/p/10080801cfe03f62bd4032bff8cb8607eb17e0/super_index'

def send_email(subject, content):
    if not EMAIL_USER or not EMAIL_PASS:
        return
    try:
        msg = MIMEText(content, 'html', 'utf-8')
        msg['Subject'] = subject
        msg['From'] = EMAIL_USER
        msg['To'] = NOTIFY_EMAIL or EMAIL_USER
        
        if 'qq.com' in EMAIL_USER:
            server = smtplib.SMTP_SSL('smtp.qq.com', 465)
        else:
            server = smtplib.SMTP('smtp.qq.com', 587)
            server.starttls()
        
        server.login(EMAIL_USER, EMAIL_PASS)
        server.send_message(msg)
        server.quit()
    except Exception as e:
        print(f"邮件发送失败: {e}")

def add_cookies(driver, cookie_str):
    """添加Cookie到浏览器"""
    driver.get('https://weibo.com')
    time.sleep(2)
    
    for item in cookie_str.split(';'):
        item = item.strip()
        if '=' in item:
            key, value = item.split('=', 1)
            # 只添加关键的认证Cookie
            if key in ['SUB', 'SUBP', 'SCF', 'PC_TOKEN', 'ALF']:
                driver.add_cookie({'name': key, 'value': value, 'domain': '.weibo.com'})
    
    driver.refresh()
    time.sleep(3)

def do_sign():
    print(f"开始签到 - {datetime.now()}")
    
    if not COOKIE_STR:
        send_email("微博签到失败", "<p>未配置Cookie</p>")
        return False
    
    chrome_options = Options()
    chrome_options.add_argument('--headless')
    chrome_options.add_argument('--no-sandbox')
    chrome_options.add_argument('--disable-dev-shm-usage')
    chrome_options.add_argument('--disable-gpu')
    chrome_options.add_argument('--window-size=1920,1080')
    
    driver = None
    try:
        driver = webdriver.Chrome(options=chrome_options)
        
        # 添加Cookie
        add_cookies(driver, COOKIE_STR)
        
        # 访问超话页面
        print(f"访问: {TOPIC_URL}")
        driver.get(TOPIC_URL)
        time.sleep(5)
        
        # 精确查找签到按钮（通过action-type属性）
        try:
            wait = WebDriverWait(driver, 10)
            # 使用更精确的选择器
            sign_btn = wait.until(EC.element_to_be_clickable(
                (By.CSS_SELECTOR, '.btn_bed .W_btn_b[action-type="widget_take"]')
            ))
            
            btn_text = sign_btn.text
            print(f"签到按钮文字: {btn_text}")
            
            # 检查是否已签到（按钮文字为"已签到"）
            if btn_text == '已签到':
                print("今日已签到")
                send_email("微博签到提醒", "<p>今日已签到</p>")
                return True
            
            # 未签到，点击签到
            sign_btn.click()
            print("已点击签到按钮")
            time.sleep(3)
            
            # 检查签到结果
            try:
                result_btn = driver.find_element(By.CSS_SELECTOR, '.btn_bed .W_btn_b[action-type="widget_take"]')
                new_text = result_btn.text
                print(f"签到后按钮文字: {new_text}")
                
                if new_text == '已签到':
                    print("✅ 签到成功!")
                    send_email("微博签到成功", f"<p>签到成功！{datetime.now()}</p>")
                    return True
                else:
                    print(f"签到后按钮未变为'已签到'")
                    send_email("微博签到失败", f"<p>点击后按钮文字: {new_text}</p>")
                    return False
                    
            except Exception as e:
                print(f"检查结果失败: {e}")
                send_email("微博签到异常", f"<p>点击后无法确认状态: {e}</p>")
                return False
                
        except Exception as e:
            print(f"未找到签到按钮: {e}")
            send_email("微博签到失败", f"<p>未找到签到按钮，可能页面结构已变化</p>")
            return False
            
    except Exception as e:
        print(f"错误: {e}")
        send_email("微博签到异常", f"<p>{e}</p>")
        return False
    finally:
        if driver:
            driver.quit()

if __name__ == '__main__':
    do_sign()
```

## 四、配置要点

### GitHub Secrets 配置

| Secret         | 说明                           | 示例                       |
| -------------- | ------------------------------ | -------------------------- |
| `WEIBO_COOKIE` | 微博登录 Cookie（含 SUB 字段） | `SUB=_2A25Eu...; SUBP=...` |
| `EMAIL_USER`   | 发件邮箱                       | `123456@qq.com`            |
| `EMAIL_PASS`   | 邮箱授权码（非登录密码）       | `abcdefghijklmn`           |
| `NOTIFY_EMAIL` | 收件邮箱                       | 可留空，默认用发件邮箱     |

### 获取微博 Cookie
1. 浏览器登录 https://weibo.com
2. F12 → Network → 刷新页面
3. 找到任意请求，复制 Request Headers 中的 Cookie 值
4. 特别确保包含 `SUB` 字段

![alt text](/img/code/微博超话自动签到/image.png)
### 获取邮箱授权码
- **QQ邮箱**：设置 → 账户 → POP3/IMAP服务 → 生成授权码
- **163邮箱**：设置 → POP3/SMTP/IMAP → 开启服务并获取授权码
